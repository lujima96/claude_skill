# Low-Poly Mobile Scout

This is the second workflow example project. It exists to prove the framework can handle a different production profile without rewriting the core pipeline.

## Current State

- Workflow selected: `low-poly-mobile-character`.
- Status: planning example only.
- Target engine: Godot.
- Target platform: mobile.
- Main differentiator: strict budgets and simple deformation scope.

## Required Next Artifacts

- Character brief.
- Reference board.
- Stage task cards.
- Handoffs.
- Specialist reviews.
- Asset manifest.
- Godot validation report.

## Reused Contracts

This project uses the same templates, validators, Character Director, QA Auditor, and Godot adapter as the stylized biped workflow.
